# 先看这里

这份交付包的目标是让甲方先验证五个独立组件本身的功能：

- Tree
- Chord
- Prism
- Scroll
- List

当前版本不包含组件间联动逻辑。组件间联动效果已经放在交互规划文档中，后续由甲方确认需要哪些交互后再实现。

## 推荐阅读顺序

1. 先读本文件，确认交付包里有什么。
2. 只想看效果：读 `docs/tutorials/01-打开-demo.md`。
3. 要接入普通 HTML 页面：读 `docs/tutorials/02-原生HTML调用.md`。
4. 要接入 React 工程：读 `docs/tutorials/03-React调用.md`。
5. 要替换成甲方自己的数据：读 `docs/tutorials/04-替换数据.md`。
6. 需要精确字段/API：读 `docs/data-contract/input-format.md` 和 `docs/component-api/`。

## 目录说明

```text
GeneticPrism-components/
  src/                 组件源码、通用建模函数、数据适配函数
  instances/
    reference/         reference 数据和 demo
    client/            client 数据和 demo
  docs/                调用说明、数据格式、PDF 文档、交互规划
  example/             源系统交互示例，仅用于参考
```

## 甲方最常用入口

原生 HTML demo：

```text
instances/reference/demos/
instances/client/demos/
```

React demo：

```text
instances/reference/demos-react/
instances/client/demos-react/
```

组件源码：

```text
src/components/
```

数据建模入口：

```text
src/core/index.js
src/adapters/index.js
```

## 交付边界

本包中 `src/` 只保留组件自身功能代码，不包含 integrated demo、combinations demo、全局 coordinator 或组件间联动入口。

`example/` 中保留源系统文件，是为了让甲方查看 legacy 效果。单组件 demo 和 React demo 不依赖 `example/prismviz.js`。
