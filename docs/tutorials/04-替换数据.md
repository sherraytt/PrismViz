# 替换数据

本教程用于把 demo 数据替换成甲方自己的数据。

## 推荐数据文件

每个数据实例建议拆成六个 JSON 文件：

```text
data/
  slice_definitions.json
  slice_hierarchy.json
  entities.json
  relations.json
  display_info.json
  component_options.json
```

reference 和 client demo 当前都采用这种拆分方式。

## 字段含义

`slice_definitions.json`：分类、主题、产业环节等切片定义。

`slice_hierarchy.json`：切片层级，用于 Tree。

`entities.json`：点数据。

`relations.json`：边数据。

`display_info.json`：List Panel 的展示字段。建议把语义清洗后的展示结果放在这里，不要让 List Panel 直接展示全部 raw/meta。

`component_options.json`：组件级配置，例如默认 slice、颜色、布局选项。

## List Panel 展示字段

建议在 `display_info.json` 中提前整理节点和边要展示的字段。

节点：

```json
{
  "entityInfoById": {
    "149048320": {
      "title": "北京电科智芯科技有限公司",
      "subtitle": "110100",
      "metrics": {
        "注册资本(万元)": 5000
      },
      "attributes": {
        "法人": "张鹏",
        "状态": "存续",
        "成立年份": 2020
      }
    }
  }
}
```

边：

```json
{
  "relationInfoById": {
    "rel_001": {
      "title": "北京电科智芯科技有限公司 -> 电科智芯有限公司",
      "subtitle": "投资关系",
      "attributes": {
        "持股比例": "100.0%",
        "投资金额(万元)": 3000
      }
    }
  }
}
```

## 数据加载

原生 demo 中可以参考：

```text
instances/reference/demo-support/reference-data.js
instances/client/demo-support/client-data.js
```

正式接入时，甲方可以从后端接口获取同样结构的数据，然后调用：

```js
import {buildComponentModels} from "./src/core/index.js";

const model = buildComponentModels({
  normalized: true,
  slices,
  hierarchy,
  entities,
  relations,
  displayInfo,
  config,
});
```

## 替换检查清单

- 每个 `entity.id` 唯一。
- 每条 `relation.source` 和 `relation.target` 都能在 `entities` 中找到。
- 每个 `entity.primarySliceId` 都能在 `slice_definitions` 中找到。
- `sliceWeights` 中的 slice id 都能在 `slice_definitions` 中找到。
- `display_info` 中的 id 和点边 id 对得上。
- 如果同一对点存在多条关系，可先在数据适配阶段聚合，再传给组件。
- Scroll 容器需要固定高度，否则页面会被右侧详情框撑开或出现半屏显示。
